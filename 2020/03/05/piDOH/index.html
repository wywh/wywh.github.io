树莓派上搭建本地的 DNS 服务器，提高局域网设备网络访问速度，为下一步将树莓派开启热点，作为网关运行透明代理服务作准备

基本原理

Linux 的 DNS 服务器由 /etc/resolv.conf 文件指定，Ubuntu 18.04+ 的 DNS 和桥接等功能默认由用户在 netplan 设置后由 systemd-resolved 自动管理，这个程序会在 127.0.0.53:53 运行一个 DNS 服务并在每次开机时修改 resolv.conf，用户不能直接修改 resolv.conf 来改变 DNS 服务器。完成后的效果：

  • SmartDNS 监听 127.0.0.1:53 和树莓派在局域网上的地址
  • 用 netplan 将本地 DNS 服务器指向 127.0.0.1:53
